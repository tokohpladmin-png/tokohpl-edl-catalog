# Product Images Folder Rule

V51 uses the Cloudinary folder `product-images`.

Product image URLs are generated by numeric design code only.

Examples:
- WY 4256X -> https://res.cloudinary.com/varindo/image/upload/product-images/4256.webp
- ATP 11358M -> tries product-images/11358.webp, then product-images/1358.webp
- WYA 15291MT -> tries product-images/15291.webp, then product-images/5291.webp

The website tries:
1. product-images/[number].webp
2. product-images/[number].jpg
3. product-images/[number].png
4. old legacy Cloudinary generated paths as fallback

No Cloudinary Admin API is required.
